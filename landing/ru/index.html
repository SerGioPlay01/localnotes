История создания

Почему появился Local Notes

Всё началось с простого разочарования — найти приложение для заметок, которое действительно приватно, быстро и бесплатно.

Проблемы существующих приложений
☁️
Всё синхронизируется с сервером. Notion, Evernote, Google Keep — ваши заметки хранятся на чужом компьютере. Одна утечка — и данные раскрыты.
💳
Шифрование стоит денег. Большинство приложений прячут настоящее шифрование за платной подпиской. Приватность не должна быть премиум-функцией.
📋
Везде нужна регистрация. Чтобы написать заметку, нужен email, пароль и согласие с политикой, разрешающей использовать ваши данные.
🌐
Нет офлайн-поддержки. Большинство веб-приложений бесполезны без интернета. Ваши заметки должны быть доступны всегда и везде.
Начало поиска
Поиск приватного приложения для заметок
Перепробовано десятки приложений. Каждое требовало аккаунт, синхронизировалось с облаком, не имело шифрования или брало деньги за базовые функции. Ничто не казалось по-настоящему приватным.
Осознание
Браузер уже имеет всё необходимое
Современные браузеры имеют IndexedDB для локального хранения, Web Crypto API для шифрования AES-256 и Service Workers для офлайн-работы. Сервер не нужен.
Первый прототип
Создан минимальный рабочий вариант
Простой редактор, локальное хранилище, базовое шифрование. Никаких фреймворков, никаких зависимостей. Только HTML, CSS и чистый JavaScript. Работало — и было быстрым.
Развитие идеи
Теги, календарь, 12 языков, PWA
Добавлен форматированный текст, цветные теги, встроенный календарь, поиск с транслитерацией и полная локализация на 12 языков. Упаковано как PWA для нативной установки.
Сегодня — v1.1.0
Открытый исходный код и бесплатно для всех
Local Notes распространяется под лицензией MIT и будет бесплатным всегда. Цель проста: дать каждому безопасное приватное место для записей — ничего не требуя взамен.
🔒
Ответ: хранить всё в браузере
Local Notes использует ваш браузер как защищённое хранилище. Шифрование AES-256, IndexedDB, Service Worker для офлайн — всё встроено в современные браузеры. Без сервера, без аккаунта, без компромиссов.

Формат .note

Ваши данные, ваш формат

Файл .note — это портативный зашифрованный контейнер, созданный с одной целью: ваши заметки принадлежат вам, а не платформе, облаку или компании.

🔒
Рождён из проблемы приватности
Каждое популярное приложение для заметок хранит данные на удалённых серверах. Одна утечка, один запрос властей, одно изменение политики — и ваши личные мысли раскрыты. Формат .note делает это невозможным по своей природе.
📦
Что внутри файла .note
Файл .note — это самодостаточный JSON-пакет: содержимое заметки, метаданные, теги и — при включённом шифровании — шифртекст AES-256, который может расшифровать только ваш пароль. Никакого проприетарного формата, никакой привязки к вендору.
🌐
Офлайн-первый, всегда переносимый
Экспортируйте любую заметку как файл .note и носите на флешке, отправляйте через Signal или архивируйте на десятилетия. Импортируйте обратно в Local Notes на любом устройстве — без аккаунта, без интернета.
🛡️
Zero-knowledge по умолчанию
При шифровании заметки ключ никогда не покидает ваше устройство. Local Notes использует встроенный в браузер Web Crypto API — тот же стандарт, что используют банки. Даже если кто-то перехватит ваш файл .note, он увидит только шифртекст.

Частые вопросы

Ваши заметки хранятся в IndexedDB вашего браузера — локальной базе данных, встроенной в каждый современный браузер. Ничего никогда не отправляется на сервер. Ваши данные не покидают устройство.

Очистка хранилища браузера удалит ваши заметки. Рекомендуем регулярно экспортировать заметки в форматах .note, HTML или Markdown. Файлы .note можно импортировать обратно в любое время.

При включении шифрования заметки шифруются с помощью Web Crypto API, встроенного в браузер. Ключ, производный от вашего пароля, используется для шифрования каждой заметки перед сохранением. Без правильного пароля данные нечитаемы.

Local Notes намеренно работает только локально — синхронизации нет по замыслу. Для переноса заметок между устройствами экспортируйте их в файлы .note и импортируйте на другом устройстве.
